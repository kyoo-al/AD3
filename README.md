# ИИ-аналитик данных — мини-продукт с LLM-агентом

Веб-приложение: пользователь загружает таблицу (CSV/Excel), а LLM-агент
**сам** анализирует данные, выполняя Python-код через инструмент (code
interpreter), и выдаёт ключевые метрики, графики и текстовые инсайды.

Главное по заданию: модель **не перефразирует** характеристики датасета из
промпта — она работает как агент, вызывает инструмент `run_python`, получает
реальные результаты вычислений на данных и строит выводы на их основе.

## Задача

Задание 3 — мини-продукт с LLM-аналитикой.
Веб-интерфейс (Streamlit) для загрузки файла → автоматический анализ данных
LLM-агентом → метрики, графики, инсайды.

## Как это устроено (агент + интерпретатор кода)

```
Пользователь -> загружает CSV/Excel -> pandas.DataFrame (df)
        |
        v
Streamlit (app.py)
        |
        v
Агент (agent.py): цикл tool-calling
   LLM  ──(tool_call: run_python с кодом)──►  песочница (exec, df, pd, np, plt)
     ▲                                              │
     └────────── результат вычислений ◄─────────────┘
        ... повторяется, пока агент не закончит ...
        |
        v
Финальный отчёт: метрики + инсайды + графики
```

- Модель получает только **структуру** данных (размер, типы, первые строки).
- Все цифры, корреляции, агрегаты и графики агент считает сам через `run_python`.
- Инструмент описан в формате function calling; модель сама пишет код,
  приложение его выполняет и возвращает вывод обратно в модель.

Модель: `nvidia/nemotron-3-super-120b-a12b:free` через OpenRouter (поддерживает
tool calling).

## Установка и запуск

```bash
# 1. зависимости
python -m venv venv
source venv/bin/activate          # Windows: venv\Scripts\activate
pip install -r requirements.txt

# 2. ключ OpenRouter — файл .env рядом с app.py
echo "OPENROUTER_API_KEY=sk-or-v1-..." > .env

# 3. запуск
streamlit run app.py
```

Откроется браузер (по умолчанию http://localhost:8501). Загрузите
`sample_sales.csv` (или свой файл), при желании впишите цель анализа и нажмите
«Запустить ИИ-анализ».

> Модель reasoning-класса, поэтому полный анализ занимает ~1-3 минуты
> (несколько шагов вычислений). Ход работы агента виден на экране в реальном
> времени.

## Что показывает приложение

- Предпросмотр данных и базовые метрики (строки, столбцы, пропуски).
- **Ход работы агента** — каждый шаг: выполненный Python-код и его вывод
  (доказательство, что анализ реальный, а не выдуман).
- **Графики**, построенные агентом (matplotlib).
- **Выводы и инсайды** — текстовый отчёт с конкретными числами и
  рекомендациями. Отчёт можно скачать в `.md`.

## Демо-датасет

`sample_sales.csv` — 600 записей о продажах: дата, регион, категория, менеджер,
цена, количество, скидка, выручка, прибыль. На нём агент находит драйверы
выручки/прибыли, сезонность, различия по регионам и категориям, аномалии.
